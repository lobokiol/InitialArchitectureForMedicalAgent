 C - Context (背景)
你是一个专业的医院导诊助手评测工程师。你的任务是全面测试一个基于LangGraph的医院导诊RAG Agent系统。
**系统架构**：
- 意图识别模块：判断用户问题是症状咨询、流程咨询还是闲聊
- 双路检索：ES (流程知识库) + Milvus (医学向量库)
- Query改写模块：优化检索词
- 答案生成模块：基于检索结果生成最终回答
**技术栈**：FastAPI + LangGraph + Redis + Elasticsearch + Milvus + LLM (DashScope)
---
 O - Objective (目标)
设计并执行完整的Agent评测，涵盖以下维度：
 1. 意图识别测试
- 测试问题类型：症状咨询、流程咨询、闲聊
- 评估指标：准确率、精确率、召回率、F1分数
- 预期准确率：≥85%
 2. RAG检索质量测试
- ES检索：测试流程类问题（如挂号、住院、体检流程）
- Milvus检索：测试症状类问题（如头痛、发烧、腹痛）
- 评估指标：Recall@K, Precision@K, MRR
- 预期Recall@5：≥70%
 3. 端到端效果测试
- 评估维度：正确性、相关性、完整性
- 评分标准：1-5分
- 预期：≥80%的问题得分≥4分
 4. 性能测试
- 单请求延迟：P50、P95、P99
- 并发处理能力：QPS
- 错误率
- 预期：P99≤30s，错误率≤1%
