# 企业级微信医生导诊系统技术方案

## 一、系统概述

### 1.1 背景

随着移动互联网和AI技术的发展，越来越多的医院和医疗机构希望通过微信小程序/公众号为患者提供智能导诊服务。患者可以通过文字、语音、图片等方式描述症状，系统自动识别并推荐合适的科室，同时提供初步的病情分析和就医建议。

### 1.2 目标

- 实现7x24小时智能导诊服务
- 减轻医院预检分诊压力
- 提升患者就医体验和效率
- 降低误诊和挂错科室的概率

### 1.3 部署形态

```
微信用户 → 微信小程序/公众号 → 后端服务 → AI推理引擎 → 数据服务
```

### 2.2 技术选型

| 组件 | 技术方案 | 说明 |
|------|---------|------|
| **后端框架** | FastAPI / Spring Boot | 高性能API服务 |
| **Agent框架** | LangGraph / LangChain | 多轮对话编排 |
| **LLM** | Qwen / GPT-4 / DeepSeek | 症状提取、追问生成 |
| **向量模型** | text-embedding-v2 / BGE | 语义匹配 |
| **知识图谱** | Neo4j | 症状-疾病-科室关系 |
| **向量检索** | Milvus / Faiss | 高效相似度检索 |
| **流程检索** | Elasticsearch | 医院流程文档检索 |
| **数据库** | PostgreSQL | 结构化数据存储 |
| **缓存** | Redis | 会话状态、热点缓存 |
| **部署** | Docker + K8s | 容器化编排 |

## 三、数据需求

### 3.1 知识图谱数据

#### 3.1.1 数据内容

| 实体类型 | 示例 | 数量级 |
|---------|------|--------|
| 症状 | 头痛、发热、腹痛、咳嗽 | 3,000+ |
| 疾病 | 感冒、肺炎、胃炎、高血压 | 8,000+ |
| 科室 | 消化内科、心内科、神经内科 | 80+ |
| 药品 | 阿莫西林、布洛芬、感冒灵 | 10,000+ |
| 检查 | 血常规、CT、核磁共振、B超 | 1,000+ |

#### 3.1.2 关系数据

| 关系类型 | 说明 | 示例 |
|---------|------|------|
| 症状-→疾病 | 症状可能导致疾病 | 咳嗽→肺炎 |
| 疾病-→科室 | 疾病对应科室 | 高血压→心内科 |
| 症状-→科室 | 症状直接推荐科室 | 胸闷→心内科 |
| 症状-→症状 | 伴随症状 | 头痛→伴恶心 |
| 疾病-→检查 | 疾病需要检查 | 肺炎→胸片 |
| 疾病-→药品 | 疾病常用药品 | 感冒→感冒灵 |

#### 3.1.3 数据来源

| 来源 | 用途 | 获取方式 |
|------|------|---------|
| CM3KG | 症状-疾病-科室基础数据 | 开源数据集 |
| 医学教材 | 疾病定义、诊疗指南 | 采购/自建 |
| 医院HIS | 真实科室分布、医生专长 | API对接 |
| 临床指南 | 疾病诊疗规范 | 采购/网络采集 |
| 药品数据库 | 药品说明书、适应症 | 公开API |

### 3.2 对话数据

#### 3.2.1 数据类型

| 类型 | 用途 | 获取方式 |
|------|------|---------|
| 问诊对话 | 训练/评测 | 脱敏历史数据/自建 |
| FAQ问答 | 直接检索 | 医院提供/采集 |
| 相似病例 | RAG检索 | EMR脱敏数据 |

#### 3.2.2 对话意图分类

| 意图 | 示例 | 处理方式 |
|------|------|---------|
| 症状描述 | "我肚子疼" | 走症状提取流程 |
| 科室咨询 | "感冒挂什么科" | 直接推荐科室 |
| 流程咨询 | "怎么挂号" | 流程文档检索 |
| 预约挂号 | "我要挂号" | 转人工/对接HIS |
| 病情咨询 | "这个病严重吗" | 风险评估+建议 |
| 闲聊 | "你好" | 友好回应 |

### 3.3 文档数据

| 类型 | 内容 | 用途 |
|------|------|------|
| 科室介绍 | 各科室擅长、医生排班 | 科室推荐依据 |
| 就医流程 | 挂号、住院、检查流程 | 流程咨询 |
| 医保政策 | 报销流程、注意事项 | 医保咨询 |
| 健康科普 | 疾病预防、护理知识 | 健康宣教 |

### 3.4 患者数据（如有）

| 数据 | 用途 | 隐私等级 |
|------|------|---------|
| 历史就诊记录 | 个性化推荐 | 高 |
| 过敏史 | 用药提醒 | 高 |
| 既往病史 | 辅助诊断 | 高 |

## 四、模型需求

### 4.1 大语言模型（LLM）

#### 4.1.1 推荐模型

| 场景 | 推荐模型 | 替代方案 |
|------|---------|---------|
| 症状提取 | Qwen-Turbo / GPT-4 | DeepSeek |
| 追问生成 | Qwen-Turbo | GPT-3.5 |
| 答案生成 | Qwen-Max / GPT-4 | DeepSeek |
| 图片分析 | Qwen-VL-Max | GPT-4V |

#### 4.1.2 模型能力要求

- ✅ 支持中文
- ✅ 支持函数调用（Function Calling）
- ✅ 支持上下文记忆
- ✅ 支持JSON输出
- ✅ 响应速度 < 3秒

### 4.2 向量模型

| 模型 | 用途 | 维度 |
|------|------|------|
| text-embedding-v2 | 症状语义匹配 | 1536 |

```
                ▼
```

```
                │
                ▼
┌─────────────────────────────────────────┐
│ 路径2: 疾病推理                         │
│ 发热 → 感冒 → 呼吸内科 (0.75)          │
│ 发热 → 肺炎 → 呼吸内科 (0.80)          │
└─────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────┐
│ RAG检索                                 │
│ 相似病例 → 科室参考                     │
└─────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────┐
│ 融合评分                                 │
│ 综合分 = 0.6×KG + 0.4×RAG              │
│ 呼吸内科: 0.6×0.80 + 0.4×0.75 = 0.78  │
└─────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────┐
│ 置信度判断                               │
│ ≥ 0.8: 直接推荐科室                     │
│ 0.5-0.8: 推荐+建议追问                  │
│ < 0.5: 追问补充症状                     │

```

## 九、总结

### 9.1 核心能力

- ✅ 智能症状识别与标准化
- ✅ 基于知识图谱的科室推荐
- ✅ 多轮对话与动态追问

- LangGraph 多轮对话状态机
- KG + RAG 混合推理

- 所有AI建议仅供参考，不能替代医生诊断
- 危急症状必须立即告警并建议就医

- 需要持续优化和维护知识库
