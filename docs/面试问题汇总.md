## 一、LangGraph / LangChain 相关 (15题)

### 1. LangGraph 和 LangChain 的区别是什么？为什么选择 LangGraph？
**LangChain 是工具库（链式调用），LangGraph 是状态机（图结构），LangGraph 适合复杂多轮对话的确定性流程控制。**

### 2. LangGraph 中的 StateGraph 是如何工作的？请解释状态管理机制
**StateGraph 是有向图 + 共享状态对象，节点读写同一个 TypedDict，边决定流转路径，状态自动传递累积。**

### 3. LangGraph 如何实现多轮对话的状态持久化？
**通过 Checkpoint 快照 + thread_id 存储到 SQLite/PostgreSQL，按会话ID恢复历史状态，实现断点续传。**

### 4. LangGraph 中的 Checkpoint 功能是什么？用于什么场景？
**节点执行后的状态快照，用于对话恢复、时间旅行调试、人工中断后恢复、多轮上下文保持。**

### 5. LangGraph 条件边 (conditional edge) 是如何实现的？
**在边上定义路由函数，根据当前状态返回目标节点名，实现 if-else 动态分支（如意图路由）。**

### 6. 什么是 Human-in-the-loop？在 LangGraph 中如何实现？
**人工介入机制，用 interrupt_before/interrupt_after 在关键节点暂停，等待人工审批后继续执行。**

### 7. LangGraph 中的节点 (node) 和边 (edge) 是什么关系？
**节点是处理函数（业务逻辑），边是流转规则（执行顺序），可无条件跳转或条件路由。**

### 8. LangGraph 如何处理对话中断和恢复？
**通过 Checkpoint 保存状态 + interrupt 暂停点，用户中断后可从断点恢复，继续执行未完成任务。**

### 9. LangGraph 支持哪些消息类型？如何处理不同类型的消息？
**支持 HumanMessage、AIMessage、SystemMessage、ToolMessage，通过消息列表维护对话历史。**

### 10. LangGraph 的流式输出 (streaming) 是如何实现的？
**用 astream()/stream() 方法逐步返回 token，支持 token 级、节点级、状态级流式输出。**

### 11. LangGraph 如何实现意图路由 (intent routing)？
**在条件边中根据意图识别结果返回节点名，如 diagnosis、appointment 分别跳转到对应处理流程。**

### 12. LangGraph 和 ReAct 框架的区别是什么？
**ReAct 是固定循环（Thought-Action-Observation），LangGraph 是可编程图，支持分支、循环、并行、人工介入。**

### 13. LangGraph 中的 Tool Calling 是如何实现的？
**定义工具函数注册到图，LLM 输出工具调用请求，LangGraph 自动执行并以 ToolMessage 返回结果。**

### 14. LangGraph 如何实现多 Agent 协作？
**通过子图嵌套或 Handoff 机制，一个 Agent 可委派任务给另一个，支持 Supervisor 模式。**

### 15. LangGraph 的优缺点是什么？有什么改进空间？
**优点：状态可控、流程透明、支持中断恢复；缺点：调试困难、学习曲线陡；改进：可视化调试工具、Checkpoint 性能优化。**

## 三、知识图谱 / Neo4j 相关 (10题)

### 1. 为什么要用知识图谱？它和 RAG 有什么区别？
**知识图谱 = 结构化实体+关系+推理能力（症状→疾病→科室两跳推理），RAG = 非结构化文本检索；图谱可解释、推理强，RAG 灵活好构建，两者融合互补（项目用 RRF 融合）。**

### 2. Neo4j 中如何创建节点和关系？请写出 Cypher 语句
**创建节点：`CREATE (n:Symptom {name: "发热"})`，创建关系：`MATCH (s:Symptom {name:"发热"}), (d:Disease {name:"感冒"}) CREATE (s)-[r:leads_to {weight: 0.8}]->(d)`。**

### 3. Neo4j 的向量索引是如何工作的？
**Neo4j 向量索引使用近似最近邻（ANN）算法，将节点属性向量存储后支持相似度检索，适用于语义搜索场景。**

### 4. 如何从零构建一个医疗知识图谱？
**从医学数据库（ICD-10、CM3KG）导入症状-疾病-科室实体和关系，通过 Cypher 批量导入并设置权重（症状-疾病共现次数），构建索引。**

### 5. 知识图谱中的概率/权重是如何存储的？
**存储为关系的 weight 属性（如症状-疾病共现次数），或节点的置信度属性，查询时用于排序和筛选。**

### 6. Neo4j 查询性能慢怎么办？有哪些优化手段？
**创建索引（CREATE INDEX）、使用参数化查询、限制返回结果数量、优化 Cypher 语句（避免全表扫描）、分片处理。**

### 7. 什么是图数据库的社区检测算法？
**用于发现图中紧密连接的节点群体（如 Louvain、Label Propagation），可识别疾病群、症状群，辅助科室聚类。**

### 8. Neo4j 和 Milvus 的区别是什么？什么场景下用哪个？
**Neo4j = 图数据库（关系推理），Milvus = 向量数据库（语义检索）；图谱推理用 Neo4j，语义搜索用 Milvus，项目两者结合。**

### 9. 如何实现知识图谱的动态更新？
**通过消息队列同步：内容变更时同时删除/增加节点和关系，更新索引；或使用 Neo4j 的 MERGE 语法实现 upsert。**

### 10. 图谱中的实体消歧是怎么做的？
**基于上下文语义（LLM 判断）、属性相似度（名称/类别）、关系路径消歧；或通过实体链接到标准知识库（如 ICD-10）。**
症状--疾病--科室，疾病为标准知识库参考icd-10，颗粒度能够保障

## 四、LLM / 模型 相关 (15题)

### 1. 你用的LLM是什么？为什么选择它？
**用 Qwen2.5-7B（7B）+ Qwen2.5-72B（72B INT4）分层架构，选择原因：开源可控、成本低、国产化、7B 单卡可推理、72B INT4 能在 H20 96GB 部署。**

### 2. 为什么要做模型微调？直接用 API 不行吗？
**微调针对意图识别场景，一些特殊场景，用fewshot非常冗余，就用 LoRA 微调 7B 提升准确率到 95%+，比纯 API 调用更可控、成本更低、响应更快。**

### 3. LoRA 的原理是什么？为什么能减少训练参数量？
**LoRA 在预训练权重旁添加低秩矩阵（如 W'=W+BA），训练时只更新 A 和 B，冻结原始权重，参数量从全量降到万级（如 r=8 时约参数量=8×dim）。**

### 4. LoRA 中的 r (rank) 和 alpha 参数的作用是什么？
**r（rank）= 低秩矩阵的秩，决定可训练参数量；alpha = 缩放系数，通常设为 r 的 2-4 倍，用于调节微调权重的影响程度。**

### 5. 微调后的模型如何评估效果？
**用测试集评估槽位提取准确率、意图分类准确率、召回率；项目最终达到 95%+ 准确率。**

### 6. 4-bit 量化是什么原理？对模型效果有什么影响？
**4-bit 量化将权重从 FP16/BF16 压缩到 4bit（NF4），通过量化+反量化流程，约 1/4 显存，可在 H20 部署 72B 模型；效果略有损失但可接受。**

### 7. 什么是 PagedAttention？vLLM 为什么使用它？
**PagedAttention 将 KV Cache 分页管理，减少显存碎片，vLLM 使用它实现更高效的批处理推理，提升吞吐量 2-3 倍。**

### 8. 模型推理时，显存是如何分配的？
**显存主要用于：模型权重（FP16约144GB/72B，INT4约36GB）、KV Cache（上下文长度×batch×层数）、激活值；通过量化+分页注意力优化分配。**

### 9. 如何控制 LLM 的幻觉 (hallucination)？
**RAG 检索相关文档作为上下文、提示词约束输出格式、设置 temperature=0 减少随机性、用 Function Calling 限制输出范围。**
强调答案来自于输入的资料，t设置为0，如果没有搜到相关资料，则输出为空，指明参考文档来自于哪里

### 10. 什么是 Function Calling？你的项目用了吗？
**Function Calling = LLM 输出结构化函数调用请求；项目用于 tool_calling 节点，调用 emergency_check、department_by_symptom 等 MCP 工具。**
kg+rag工具检索
### 11. LLM 的温度 (temperature) 参数有什么用？
**temperature 控制输出随机性，0=确定性输出（适合任务型），0.7=创造性输出（适合对话），导诊系统用 0.1-0.3。**
随机性参数，越大越随机，越小越稳定

### 12. 如何让 LLM 输出结构化 JSON？
**用 JSON Schema 约束（with_structured_output）或提示词指定 JSON 格式 + 后处理解析；项目用 Pydantic 定义 ExtractedSymptoms 结构。**
用schema约束，制定json格式， Few-shot 指示 JSON 结构，vllm
### 13. 什么是 Chain-of-Thought (CoT) 提示？
**CoT 在提示词中加"请一步步思考"，让 LLM 输出推理过程，提升复杂任务（数学、逻辑）准确率。**
项目没用，但我理解是推理过程展示出来，让模型学习
### 14. 小模型 (7B) 和大模型 (70B) 怎么选？
**7B：单卡可推理、成本低，适合意图识别、槽位填充等简单任务；70B：复杂推理、多跳思考，适合最终回答生成；项目用 7B+72B 分层。**
简单任务用7b，复杂推理，多跳，最终回答兜底用72b
### 15. 模型推理延迟高怎么办？
**量化（INT4/INT8）、vLLM 优化、批处理、缓存（热点结果）、减少上下文长度、异步调用；项目用 7B 承担实时任务，72B 异步生成。**
先检查是否开启了 vLLM 的 Prefix Caching 和 FlashAttention-2提示吞吐量，这是目前性价比最高的优化手段。同时，确保前端已实现 Stream 展示，这能解决 80% 的用户“延迟感”投诉，提供stream功能，7B 承担实时任务，promt缓存常用，减少上下文，加配置gpu

主槽位和附属槽位，通过输入 -- 清洗 -- embeding --实体转换--规则校验（否定词汇）-- llm提取 -- 症状 -- kg校验。进行填充
5. 多轮对话中，如何判断用户是否说完了？

轮询，根据槽位信息进行追问，通过模型生成个性化问题。直到槽位满足要求或置信度大于0.8
7. 如何处理用户的否定表达（如"我不发烧"）？
槽位会有否定症状信息，大模型会通过关键字，否定词汇匹配，检索的时候，去掉这个症状
8. 意图分类有哪些方法？规则 vs LLM vs 微调？
llm+微调，单llm或规则无法处理隐喻，紧急，政治敏感的场景
9. 如何处理意图识别置信度低的情况？
可以先回答固定问题，哪里疼，疼多久，什么症状，置信度低的化，可以追问上述问题，有闲聊选型，可以拒绝回答，并提示回答哪里疼
10. 什么是对话状态跟踪 (Dialogue State Tracking)？
追踪对话中用户提到的症状、意图、槽位等信息，项目中通过 LangGraph 的 State 状态机 + Checkpoint 实现
追踪用户提到的症状，意图，槽位等信息，通过state状态机和checkpoint实现，可以把状态保存在数据库中，或者使用 Redis，实现下次启动保持上一个状态

## 六、后端 / 架构 相关 (15题)

### 1. 请介绍一下你的项目整体架构
**整体架构：FastAPI + LangGraph 状态机 + RAG + KG + Neo4j + Milvus + Redis，分层：接入层（API）→ 决策层（意图识别）→ 检索层（RRF融合）→ 生成层（LLM）。**

### 2. FastAPI 和 Flask 的区别是什么？为什么选 FastAPI？
**FastAPI = 异步 + 类型提示 + 自动 API 文档；Flask = 同步 + 灵活；选 FastAPI 因为原生异步性能高、自动生成 Swagger 文档、开发效率高。**

### 3. FastAPI 如何实现异步处理？
**用 async/await + Starlette 框架，路由函数加 async def，数据库/外部调用用 aiohttp/asyncpg，实现高并发。**

### 4. 你的项目如何部署？用 Docker 吗？
**用 Docker Compose 部署，支持单医院 5-10 并发，包含：API服务 + Neo4j + Milvus + ES + Redis + 模型服务。**

### 5. 数据库设计有哪些原则？
**三范式 + 适当反范式 + 索引优化 + 主键 UUID/自增 + 外键约束 + 读写分离。**

### 6. Redis 在项目中起到了什么作用？
**用于：1) 会话状态存储（thread_id → 对话状态）；2) 热点数据缓存；3) 限流/计数器。**

### 7. 如何保证 API 的安全性？
**JWT 鉴权 + HTTPS + 输入校验 + 限流 + CORS + API Key 认证。**

### 8. 接口性能慢如何排查和优化？
**排查：日志 + 监控 + Py-Spy/Profiling；优化：索引 + 缓存 + 异步 + 减少 JOIN + 量化模型。**

### 9. 什么是 API Gateway？为什么要用它？
**API Gateway = 统一入口（鉴权 + 限流 + 路由 + 监控）；用于多服务统一管理、流量控制、安全防护。**

### 10. 如何设计高可用的系统架构？
**负载均衡 + 服务冗余 + 限流降级 + 多级缓存 + 异步解耦 + 监控告警 + 自动恢复。**

### 11. 你的项目有哪些监控和告警？
**接口响应时间 + 错误率 + GPU 利用率 + 模型推理延迟 + Redis/Milvus 状态，用 Prometheus + Grafana。**

### 12. 日志是如何设计的？如何排查问题？
**分层日志（INFO/WARN/ERROR）+ trace_id 串联请求 + 结构化 JSON 日志 + ELK 收集排查。**

### 13. 前后端数据交互用的是什么格式？
**JSON 格式（application/json），流式用 SSE（text/event-stream）。**

### 14. 如何处理并发请求？
**异步（async/await）+ 连接池 + 限流 + 队列缓冲 + 无状态服务 + 负载均衡。**

### 15. 你的项目如何做缓存？
**Redis 缓存：会话状态 + 热点检索结果 + 模型推理结果 + 科室推荐结果，TTL 分级设置。**

## 七、场景题 (10题)

### 1. 用户说"我肚子疼"但不说具体位置，你怎么处理？
**触发追问："请问是上腹部还是下腹部？左边还是右边疼？"基于槽位填充逻辑，引导用户补充位置信息。**

### 2. 用户问了一个医学禁忌问题（如推荐止痛药）你怎么办？
**拒绝回答 + 引导就医："这类药物需要医生处方，建议您到医院就诊，由医生根据情况开具。"**

### 3. 知识库没有匹配到答案时，怎么回复用户？
**"抱歉，暂时未找到相关信息，建议您描述更多症状，或直接前往医院导诊台咨询。"**

### 4. 同时有多个人问不同的问题，系统如何处理？
**每个请求独立处理（thread_id 隔离），通过 LangGraph 多实例/多进程部署支持并发，系统无状态设计。**

### 5. 用户打着打着字就停了，你怎么办？
**检测输入超时 → 发送追问确认："您还在吗？请描述您的症状。"超时无响应 → 保留状态，等待用户下次继续。**

### 6. 用户问的问题超出系统范围，怎么引导？
**"这个问题我无法回答，医院导诊台可以帮您详细解答，建议您前往现场咨询。"**

### 7. 如何处理危急症状（如胸痛、呼吸困难）？
**CRITICAL_RISKS 规则 + Neo4j 紧急规则 → 立即触发告警 → "您描述的症状危急，请立即拨打120或前往急诊！"**

### 8. 用户要求投诉或退款，你怎么办？
**"抱歉给您带来不便，您可以通过 XX 渠道进行投诉/退款，我来帮您转接。"转人工处理。**

### 9. 系统回答错了科室，有什么兜底策略？
**1) 置信度低于阈值时追问确认；2) 返回多个科室建议；3) 提示"仅供参考，以医院分诊为准"；4) 收集错误反馈用于优化。**

### 10. 如果要对接医院 HIS 系统，你会怎么设计？
**通过 API 网关 + 标准接口（HL7/FHIR）+ 消息队列异步同步，患者信息加密传输，权限控制到接口级别。**

### 2. 项目中遇到的最大技术挑战是什么？怎么解决的？

最大的技术挑战是**意图识别的准确率问题**。

**问题表现：**
- 原始意图识别准确率只有 70%
- 对混合意图（同时有症状和流程问题）识别效果很差
- 对否定表达（如"我不发烧"）处理不好

**解决方案：**
1. 构建了 351 条全面的测试数据集，覆盖 9 种场景
2. 设计了 LLM-as-Judge 评估方案进行自动化评估
3. 引入 LoRA 微调方案优化意图识别模型
4. 添加了独立的危急症状检测模块作为安全兜底

**效果：** 通过测试驱动开发，逐步优化，意图识别准确率显著提升
