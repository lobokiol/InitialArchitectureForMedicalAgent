## 八、项目经验 (10题)

### 1. 请介绍你最满意的一个项目

我最满意的是**医院智能导诊系统**这个项目。这是一个基于 LangGraph + RAG + 知识图谱的 AI Agent 智能体，用于帮助患者进行症状问诊和科室推荐。

**项目亮点：**
- 完整的多轮问诊系统，通过槽位填充逐步收集患者症状信息
- 融合知识图谱 (Neo4j) 和向量检索 (Milvus + ES) 的混合 RAG 架构
- MCP 协议统一调度各数据源
- 支持意图识别、症状提取、科室推荐、答案生成完整流程
- 包含 LoRA 微调方案用于意图识别优化

**技术栈：** LangGraph + FastAPI + Neo4j + Milvus + Elasticsearch + Redis + PostgreSQL

---

### 2. 项目中遇到的最大技术挑战是什么？怎么解决的？

最大的技术挑战是**意图识别的准确率问题**。

**问题表现：**
- 原始意图识别准确率只有 70%
- 对混合意图（同时有症状和流程问题）识别效果很差
- 对否定表达（如"我不发烧"）处理不好

**解决方案：**
1. 构建了 351 条全面的测试数据集，覆盖 9 种场景
2. 设计了 LLM-as-Judge 评估方案进行自动化评估
3. 引入 LoRA 微调方案优化意图识别模型
4. 添加了独立的危急症状检测模块作为安全兜底

**效果：** 通过测试驱动开发，逐步优化，意图识别准确率显著提升

### 7. 如果让你重新做这个项目，会有什么不同？

1. **更早引入测试驱动开发**
   - 先定义清楚评估指标和测试用例
   - 再开始写代码

2. **预训练数据准备更充分**
   - 意图识别需要更多高质量标注数据
   - 考虑使用 LLM 生成合成数据 + 人工审核

3. **增加预处理节点**
   - 在意图识别前增加文本标准化
   - 减少 LLM 的理解错误

4. **更完善的监控体系**
   - 接入 Langfuse 进行效果追踪
   - 实时监控各项业务指标

5. **安全合规先行**
   - 早期就考虑医疗合规要求
   - 设计完善的免责声明机制
