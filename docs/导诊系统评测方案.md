## 二、核心评测指标

### 2.1 科室匹配准确率（最重要）

```
准确率 = 推荐科室与医生标注一致的样本数 / 总样本数 × 100%

目标: > 95%
```

**黄金测试集构建：**

| 项目 | 内容 |
|------|------|
| 样本量 | 200-500 条真实患者主诉 |
| 来源 | 医院临床医生提供 |
| 标注 | 医生人工标注"标准答案"（应挂科室） |

**示例：**
```
主诉: "我左边肚子疼，还拉肚子"
标注: 科室=消化内科, 风险等级=普通

主诉: "胸口像大石头压着，喘不过气"
标注: 科室=急诊科/心内科, 风险等级=危急
```

### 2.2 症状提取准确率

```
症状提取准确率 = 正确提取的症状数 / 实际症状总数 × 100%

否定症状识别率 = 正确识别的否定症状数 / 实际否定症状总数 × 100%
```

**测试用例：**

| 输入 | 预期输出 | 检查点 |
|------|----------|--------|
| "我头痛，不发烧" | symptoms=["头痛"], negative=["发热"] | 否定识别 |
| "头壳痛，肚子疼" | symptoms=["头痛", "腹痛"] | 俗语转换 |
| "胸口闷闷的，没有咳嗽" | symptoms=["胸闷"], negative=["咳嗽"] | 多症状+否定 |

### 2.3 危急症状识别率

```
危急识别率 = 正确识别的危急样本数 / 危急样本总数 × 100%
目标: 100% (零漏报)

误报率 = 误报为危急的正常样本数 / 正常样本总数 × 100%
目标: < 5%
```

**边界测试用例：**

| 测试场景 | 输入 | 预期结果 |
|----------|------|----------|
| 危急重症 | "胸口像大石头压着，喘不过气" | 触发红色预警 |
| 否定危急 | "我没有吐血，就是头疼" | 不触发危急 |
| 意图干扰 | "你们医院食堂几点开饭？" | 识别为非导诊意图 |

## 三、评测方法

### 3.1 自动化评测（RAGAS）

```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_precision,
    context_recall
)

# 准备测试集
test_data = [
    {
        "question": "我头痛还拉肚子挂什么科",
        "answer": "建议挂消化内科...",
        "contexts": ["检索到的文档1", "检索到的文档2"],
        "ground_truth": "消化内科"
    },
    ...
]

# 执行评估
result = evaluate(
    dataset=test_data,
    metrics=[faithfulness, answer_relevancy, context_precision, context_recall]
)
```

### 3.2 人工抽检

| 项目 | 方法 |
|------|------|
| 抽样 | 每周随机抽 100 条真实对话 |
| 审核 | 临床医生人工审核推荐科室是否正确 |
| 反馈 | 错误案例归档用于模型优化 |

### 3.3 A/B 测试

```
版本A: 当前系统 (7B INT4 + KG)
版本B: 基线系统 (纯规则/纯LLM)

对比指标:
- 科室匹配准确率
- 用户满意度
- 平均对话轮数
```

## 四、性能压测

### 4.1 Locust 压测脚本

```python
from locust import HttpUser, task, between

class TriageUser(HttpUser):
    wait_time = between(1, 3)
    
    @task(3)
    def chat_normal(self):
        self.client.post("/chat", json={
            "user_id": "load_test",
            "message": "我肚子疼"
        })
    
    @task(1)
    def chat_complex(self):
        self.client.post("/chat", json={
            "user_id": "load_test",
            "message": "我头痛三天了，还伴有恶心想吐的症状，不发烧"
        })
    
    @task(1)
    def chat_emergency(self):
        self.client.post("/chat", json={
            "user_id": "load_test",
            "message": "胸口疼，喘不过气"
        })
```

**压测命令：**
```bash
locust -f locustfile.py --headless -u 20 -r 2 -t 60s --host http://localhost:8080
```

### 4.2 性能目标

| 指标 | 目标 | 告警阈值 |
|------|------|----------|
| 首字延迟 (TTFT) | < 300ms | > 500ms |
| 端到端延迟 | < 1s | > 2s |
| 并发承载 | 20 请求/秒 | 错误率 > 5% |
| GPU 显存 | < 80% | > 90% |

## 七、评测流程

### 阶段一：单机模型测试
- 测试内容：算法准确性
- 工具：RAGAS + 黄金测试集

### 阶段二：内网 RAG 测试
- 测试内容：检索质量
- 工具：RAGAS + 自建测试集

### 阶段三：全链路 E2E 测试
- 测试内容：微信/HIS 业务闭环
- 工具：端到端测试 + 人工审核

---

> **重要提示**：一定要拉着医院的临床医生做"抽查"验收，他们的反馈是项目能否结项的关键。
